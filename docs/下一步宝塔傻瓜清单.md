# 下一步要做什么（傻瓜清单）

按顺序做，做完一项就打勾。不需要会写代码。

---

## 第一部分：在你自己的电脑上（Mac）

### 步骤 1：打开环境说明文件

1. 打开 **访达（Finder）**。
2. 进入桌面上的项目文件夹：**安全大师**。
3. 再进入里面的 **`docs`** 文件夹。
4. 双击打开 **`SERVER_ENV.md`**。  
   - 若用 **Cursor**：左侧文件树里点 `docs` → `SERVER_ENV.md`。

### 步骤 2：把表格里的空填上（只填你知道的，没有就写「无」）

对着下面每一项，在文件里找到同名表格，**删掉「例：」后面的示例，换成你的真实情况**：

| 在文件里找这一节 | 你填什么 |
|------------------|----------|
| 公网 IP / 域名 | 你 App「我的」里用的那个地址，例如 `http://你的IP:3000` |
| 服务器上 API 的文件夹 | 一般是：`/www/wwwroot/safemaster-api` |
| PM2 进程名 | 一般是：`safemaster-api`（不确定就在下面「第三部分」里看 `pm2 list` 的名字） |
| 监听端口 | 一般是：`3000`（除非你改过） |
| iOS App 的 baseURL | 和 Xcode 里「我的」连的那个地址一致即可（可写同上） |

**重要：** 不要在这个文件里写 **数据库密码**、**DeepSeek 的 sk- 密钥**。只写路径、端口、进程名这类信息。

### 步骤 3：保存文件

按 **Command + S**（或菜单「文件 → 存储」）保存。

---

## 第二部分：在宝塔面板里（你的服务器网页）

### 步骤 4：上传自检脚本

1. 在电脑上找到：**安全大师** 项目里的 **`server`** 文件夹。
2. 找到文件 **`verify-remote-env.sh`**。
3. 打开 **宝塔** → **文件**。
4. 进入 **`/www/wwwroot/safemaster-api`**（和你的 `index.js` 同一个文件夹）。
5. 点 **上传**，把 **`verify-remote-env.sh`** 传上去。  
6. 确认这个文件夹里 **同时有**：`index.js`、`hazardPrompts.js`、`verify-remote-env.sh`、`config.json`、`package.json`。

### 步骤 5：在宝塔里运行自检（复制粘贴即可）

1. 宝塔里点 **终端**（或 SSH 工具连上服务器也可以）。
2. **整段复制**下面三行，粘贴到终端里，按 **回车**：

```bash
cd /www/wwwroot/safemaster-api
chmod +x verify-remote-env.sh
./verify-remote-env.sh
```

3. 等它跑完，会打印一大段文字。
4. **全选复制**这段输出（从第一行 `========` 到最后一行）。
5. 打开 **Cursor**，新建对话或继续当前对话，**粘贴**这段输出，并写一句：「这是我刚跑的自检结果，请帮我看有没有问题。」

---

## 第三部分：如果你不知道 PM2 进程名

在宝塔终端里执行（可复制）：

```bash
pm2 list
```

看 **name** 那一列，例如 `safemaster-api`，把这个名字填回 **`SERVER_ENV.md`** 里对应格子。

---

## 做完之后你得到了什么

- **`SERVER_ENV.md`**：以后跟 AI 说「看我 `docs/SERVER_ENV.md`」，少解释很多遍宝塔。  
- **`verify-remote-env.sh`**：一出问题就先跑一遍，把结果贴给 AI，比只说「报错了」快很多。
