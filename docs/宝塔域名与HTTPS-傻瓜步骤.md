# 宝塔：域名 + HTTPS（Let’s Encrypt）傻瓜步骤

做完后，App 可把 API 根地址从 **`http://IP:3000`** 换成 **`https://你的域名`**（无需用户记端口）。

> **备案说明**：域名解析、在宝塔里建站、申请证书，多数情况下**可以先做**。若域名**未备案**，部分云厂商会**限制 80/443 对外访问**或提示备案——以你 **域名注册商 / 云服务器控制台** 的提示为准；若证书申请失败，先看是否被「未备案拦截」。

---

## 你需要先有的东西

- [ ] 已购买域名（如 `example.com`），且能登录 **DNS 管理**（阿里云/腾讯云/Cloudflare 等）。
- [ ] 知道 **云服务器公网 IP**（与现在用 `http://IP:3000` 访问的是同一台）。
- [ ] 宝塔能打开，且 **`safemaster-api` 已在跑**（本机 `127.0.0.1:3000` 有响应）。

---

## 第一步：域名解析到服务器 IP

在 **域名 DNS 管理** 里添加记录（名称按你习惯，常见两种）：

| 类型 | 主机记录 | 记录值 |
|------|----------|--------|
| **A** | **@**（根域名） | 你的**服务器公网 IP** |
| **A** | **www**（可选） | 同上 |

保存后等待生效，一般 **几分钟～几小时**。  
本机可先测（把 `api.你的域名.com` 换成你的真实域名或子域）：

```bash
ping api.你的域名.com
```

应能解析到你的 **公网 IP**（或先 `nslookup api.你的域名.com`）。

> **建议**：API 单独用子域，例如 **`api.example.com`**，与将来官网 `www` 分开。在 DNS 里加一条 **A 记录**：主机记录 `api`，值填 **服务器 IP**。

---

## 第二步：宝塔放行 80 / 443

1. 宝塔 → **安全**（或 **防火墙**）。  
2. 放行端口：**80**、**443**（若未放行，Let’s Encrypt 验证会失败）。  
3. 云厂商控制台（阿里云/腾讯云等）**安全组**里，同样放行 **入站 80、443**（与 22 一样在控制台配）。

---

## 第三步：在宝塔里「添加站点」（用域名）

1. 宝塔 → **网站** → **添加站点**。  
2. **域名**填：`api.你的域名.com`（或你解析用的那个完整域名）。  
3. **根目录**可先用默认（后面只做反代，不依赖这个目录放网页）。  
4. **PHP 版本**选 **纯静态** 或任意（我们主要用 **反向代理**）。  
5. 提交创建。

---

## 第四步：反向代理到本机 Node（3000 端口）

1. 打开该站点 → **设置** → **反向代理** → **添加反向代理**。  
2. **代理名称**：随意，如 `safemaster-api`。  
3. **目标 URL**：`http://127.0.0.1:3000`  
4. **发送域名**：可填 `$host` 或你的域名（按宝塔默认即可）。  
5. 保存。

浏览器访问：`http://api.你的域名.com/health`  
应看到与 `http://服务器IP:3000/health` **相同 JSON**。

若打不开：检查 **PM2** 是否在跑、`curl http://127.0.0.1:3000/health` 在 SSH 是否正常。

---

## 第五步：申请 SSL 证书（Let’s Encrypt）

1. 该站点 → **设置** → **SSL**。  
2. 选择 **Let’s Encrypt**（或宝塔「**宝塔 SSL**」里的免费证书）。  
3. **域名**勾选你的 `api.xxx.com`。  
4. 邮箱填真实邮箱 → **申请**。  
5. 成功后打开 **强制 HTTPS**（HTTP 自动跳 HTTPS）。

浏览器访问：`https://api.你的域名.com/health`  
应正常返回 JSON，且地址栏有 **锁**。

> 若申请失败：常见是 **80 端口未放行**、**域名未解析到本机**、或 **未备案被云厂商拦截**——按报错在宝塔/云厂商文档里查。

---

## 第六步：改 iOS App 里的 API 地址

1. 用 Xcode 打开工程。  
2. 打开 **`SafeMasterAPIConfiguration.swift`**。  
3. 把 **`productionBaseURL`**（或当前的线上地址）改为：

   **`https://api.你的域名.com`**

   **不要**带末尾斜杠 `/`，**不要**带 `:3000`（HTTPS 默认 443，已由 Nginx 反代到 3000）。

4. **编译安装到真机**，在「我的」里再同步一次，测「刷新云端次数」「开始排查」等。

---

## 第七步：验收清单

- [ ] `https://你的域名/health` 浏览器可打开，`dbConnected` 等与线上一致。  
- [ ] `curl -s https://你的域名/health` 在任意电脑可通（可选）。  
- [ ] App **仅使用 HTTPS**，不再写死 `http://IP:3000`（避免 iOS ATS 问题）。  
- [ ] 旧链接 `http://IP:3000` 可保留作应急，正式用户走域名。

---

## 可选：仅允许通过域名访问（更安全）

在宝塔 **Nginx 配置**里可限制：只允许 `Host` 为你的域名；或关闭外网直连 **3000**（安全组里去掉 3000 外网，只留 80/443）。**改前请确认反代已稳定**，避免把自己锁在外面。

---

## 与备案的关系（再强调）

- **解析、建站、证书**：多数流程**不依赖**备案号是否下发。  
- **若某一步被提示「需备案」**：暂停对外访问，等备案通过后再开 80/443，或咨询云厂商客服。

---

完成后可把 **`docs/SERVER_ENV.md`** 里「公网 / 域名 / HTTPS」几行填好，方便以后排错。
