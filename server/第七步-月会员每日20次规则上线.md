# 第七步：月会员（¥48）+ 每日20次分析规则

目标：把旧的 `credits` 扣次模型升级为：

- 月会员有效期内可用（当前先提供开发联调续期接口，后续接 IAP 验票）
- 每日分析上限 20 次（北京时间 00:00 重置）
- 报告生成不限次（不消耗分析配额）

---

## 1）先升级数据库字段（只需一次）

在宝塔 MySQL 执行（数据库：`safemaster`）：

```sql
ALTER TABLE users ADD COLUMN plan_status VARCHAR(16) NOT NULL DEFAULT 'inactive' COMMENT 'inactive/active';
ALTER TABLE users ADD COLUMN plan_expires_at DATETIME NULL COMMENT '会员到期时间';
ALTER TABLE users ADD COLUMN daily_limit INT NOT NULL DEFAULT 20 COMMENT '每日可用分析次数';
ALTER TABLE users ADD COLUMN daily_used INT NOT NULL DEFAULT 0 COMMENT '当日已用次数';
ALTER TABLE users ADD COLUMN daily_quota_date CHAR(10) NOT NULL DEFAULT '' COMMENT '配额日期(Asia/Shanghai, YYYY-MM-DD)';
ALTER TABLE users ADD COLUMN report_unlimited TINYINT(1) NOT NULL DEFAULT 1 COMMENT '报告生成是否不限次';
```

> 若提示某列已存在，可忽略该条并继续下一条。

---

## 2）上传并重启服务

上传覆盖：

- `index.js`
- `schema.sql`（用于新环境初始化）

然后重启：

```bash
cd /www/wwwroot/safemaster-api
pm2 restart safemaster-api
pm2 logs safemaster-api --lines 30
```

---

## 3）接口联调（服务器本机 curl）

### A. Apple 登录（首次会赠送 30 天会员用于联调）

```bash
curl -sS -X POST http://127.0.0.1:3000/v1/auth/apple \
  -H "Content-Type: application/json" \
  -d '{"identityToken":"curl-monthly-test"}'
```

返回体应包含：

- `accessToken`
- `subscription.active`（true）
- `subscription.dailyLimit`（20）
- `subscription.dailyRemaining`（20）

### B. 查询当前账户与每日剩余次数

```bash
curl -sS http://127.0.0.1:3000/v1/me \
  -H "Authorization: Bearer 这里填上一步accessToken"
```

### C. 手动续期（开发联调接口，后续由 IAP 验票替代）

```bash
curl -sS -X POST http://127.0.0.1:3000/v1/subscription/mock/renew \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer 这里填accessToken" \
  -d '{"days":30}'
```

---

## 4）App 侧表现

- 「我的」页面展示会员状态、到期时间、今日剩余次数、每日上限。
- 「开始排查/重新分析」每日最多 20 次。
- 报告导出不扣每日次数。

---

## 5）说明

- 当前 `v1/subscription/mock/renew` 仅为联调接口，生产建议在完成 IAP 后改为：验票成功再续期。
- 若用户提示“会员已过期或未开通”，可先调用 mock 续期接口确认流程。
