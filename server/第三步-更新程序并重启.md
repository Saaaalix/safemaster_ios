## 操作步骤

### 第 1 步：用你电脑里的新文件覆盖服务器上的旧文件

在你自己的电脑上（Mac），项目里 **`server/index.js`** 已经更新为**带第三步内容**的版本。

请你：

1. 打开本机文件夹：`安全大师` → `server` → 找到 **`index.js`**。  
2. 用 **文本编辑 / VS Code** 打开，**全选 → 复制** 全部内容。  
3. 登录 **宝塔** → **文件** → 进入 **`/www/wwwroot/safemaster-api`**。  
4. 点 **`index.js`** → **编辑** → **全选删除** → **粘贴**你刚复制的内容 → **保存**。

**成功标志**：保存后，文件开头几行有「第三步起」之类注释（和你本机一致）。

> 若你更习惯「上传」：也可以把本机 `index.js` 用宝塔 **上传** 覆盖，选 **覆盖** 即可。

---

### 第 2 步：重启 PM2（必须做，否则还是旧程序）

打开宝塔 **终端**，**复制粘贴**下面两行，**一行一行执行**：

```bash
cd /www/wwwroot/safemaster-api
```

```bash
pm2 restart safemaster-api
```

**成功标志**：终端里出现 `online` 或 `restarted` 等提示，没有大片红色报错。

---

### 第 3 步：用浏览器自测

在浏览器地址栏输入（IP 换成你的）：

```text
http://你的公网IP:3000/v1/me
```

**应看到**一段 JSON，里面有 **`"credits":5`**。

再打开（确认旧接口还在）：

```text
http://你的公网IP:3000/health
```

仍应有 **`"ok":true`**。

---

### 第 4 步（可选）：在终端里测「模拟登录」接口

浏览器**不能直接发 POST**，所以在宝塔终端执行：

```bash
curl -s -X POST http://127.0.0.1:3000/v1/auth/apple \
  -H "Content-Type: application/json" \
  -d '{"identityToken":"test123"}'
```

若返回 JSON 里有 **`accessToken`**、**`credits":5`**，说明 **POST 接口也正常**。

## 常见问题

**问：浏览器打开 `/v1/me` 还是旧的或报错？**  
→ 确认已 **保存** `index.js`，且执行了 **`pm2 restart safemaster-api`**。再执行 `pm2 logs safemaster-api` 看报错。

**问：提示找不到 `express`？**  
→ 一般不会，第一步已 `npm install`。若真有，在 `safemaster-api` 目录再执行一次：`npm install`。
