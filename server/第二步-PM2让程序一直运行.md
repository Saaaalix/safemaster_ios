# 安全大师 · 第二步：让 API 关掉终端也继续跑（PM2）

> **上一步你做到了**：浏览器能打开 `http://你的IP:3000/health`，看到 `ok:true`。  
> **这一步做完**：关掉宝塔「终端」黑色窗口后，**网址仍然能打开**，服务器重启后也可以自动再启动（可选）。

---

## 这一步做完你会得到什么？

- 你的 `index.js` 在后台**一直运行**，不用一直开着黑色命令行窗口。  
- 以后你改代码、重新部署，用几条固定命令即可（下一步会教）。

---

## 你需要什么？

- 已经完成 **第一步**，且 **`/www/wwwroot/safemaster-api`** 里仍有 `index.js` 和 `node_modules`。  
- 能打开 **宝塔 → 终端**，并且是 **root** 或能用 `sudo`。

---

## 操作步骤（请按顺序）

### 第 1 步：安装 PM2（只做一次）

在宝塔 **终端** 里，**复制粘贴下面这一行**，回车，等它跑完（约 1 分钟）：

```bash
npm install -g pm2
```

**成功标志**：最后一行没有大片红色报错；可以继续下一步。

若提示 `permission denied`，在前面加 `sudo` 再试一次：

```bash
sudo npm install -g pm2
```

---

### 第 2 步：如果之前用 `node index.js` 还在跑，先停掉

若你之前开了一个终端，里面还在跑 `node index.js`：

- 在那个窗口里按键盘 **`Ctrl + C`**（一次不行就再按一次），直到不再显示「监听 3000」之类字样。

如果已经关掉了旧窗口，**可以跳过这一步**。

---

### 第 3 步：用 PM2 启动你的程序

在宝塔 **终端** 里，**一段一段**执行（每段回车后等完成）：

**第一段：进入目录**

```bash
cd /www/wwwroot/safemaster-api
```

**第二段：用 PM2 启动（名字可以照抄）**

```bash
pm2 start index.js --name safemaster-api
```

**成功标志**：会显示一张表，里面有 **`safemaster-api`**、`online` 等字样。

**第三段：保存当前进程列表（推荐）**

```bash
pm2 save
```

---

### 第 4 步：验证（很重要）

1. **不要**再执行 `node index.js`（交给 PM2 管即可）。  
2. 在终端输入：

```bash
pm2 list
```

应能看到 **`safemaster-api`** 状态为 **`online`**。

3. 用手机或电脑浏览器再打开：

```text
http://你的公网IP:3000/health
```

仍应看到 **`"ok":true`**。

4. **现在可以关掉宝塔终端窗口**，再刷新一次 **同一个网址**。  
   - 若**仍能打开** → 第二步成功。  
   - 若打不开了 → 看本文末尾「常见问题」。

---

### 第 5 步（可选）：服务器重启后自动启动 PM2

在终端执行：

```bash
pm2 startup
```

终端里会出现 **一大段以 `sudo` 开头的提示**，**把那一整段复制下来再执行一次**（系统不同，生成的命令不一样，**必须复制你屏幕上显示的那句**）。

然后再执行：

```bash
pm2 save
```

---

## 以后你常用到的几条命令（先收藏，不必现在背）

| 你想做的事 | 在终端输入 |
|------------|------------|
| 看是否在跑 | `pm2 list` |
| 看最近日志 | `pm2 logs safemaster-api` |
| 改完代码后重启 | `cd /www/wwwroot/safemaster-api` 然后 `pm2 restart safemaster-api` |
| 临时停掉 | `pm2 stop safemaster-api` |

---

## 常见问题

**问：执行 `pm2 start` 说端口被占用？**  
→ 说明还有一个旧的 `node index.js` 在跑。先 `Ctrl+C` 停掉旧终端里的进程，或执行：  
`pm2 delete safemaster-api` 再重新 `pm2 start ...`。

**问：关掉终端后网页打不开了？**  
→ 执行 `pm2 list` 看是不是 **`stopped`**。若是，执行 `pm2 restart safemaster-api`。若根本没有这一行，说明没 `pm2 save` 或没启动成功，从 **第 3 步**重做。

**问：`npm install -g pm2` 很慢或失败？**  
→ 多等一会；或换手机热点试网络。把**最后 15 行**终端输出发给协助你的人。
